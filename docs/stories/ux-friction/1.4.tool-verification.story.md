# Story 1.4: Verificação de Ferramentas no Adversarial Review

**Epic:** UX Friction Reduction — Devflow Onboarding
**Sprint:** 1 (Curto Prazo)
**Priority:** P1 — Alta
**Effort:** 2 pontos
**Status:** Done

## Problem Statement

`adversarial-review` roda 12 vetores. Vários dependem de ferramentas externas (dependency-cruiser, jscpd, madge). Quando ausentes, retornam INCONCLUSIVE. Em modo strict/release, INCONCLUSIVE bloqueia. Usuário vê 4+ INCONCLUSIVE sem entender que é falta de instalação.

## User Story

AS a developer running adversarial review
I WANT to know which tools are missing BEFORE the review starts
SO THAT I can install them and get definitive PASS/FAIL results

## Acceptance Criteria

1. `adversarial-review` runs a pre-flight tool check before any vector executes
2. Pre-flight output shows tool status table with emoji indicators
3. Each missing tool shows exact install command (e.g., `npm install --save-dev jscpd`)
4. `--install-missing` flag auto-installs missing devDependencies
5. Vectors with missing tools that user chose to skip show as `⏭️ SKIPPED (tool not installed)` instead of INCONCLUSIVE
6. INCONCLUSIVE now only means "tool ran but couldn't determine" — not "tool missing"

## Technical Notes

- **Files:** `src/commands/adversarial-review.ts` (pre-flight + install flag)
- **New:** `src/kernel/checks/tool-verifier.ts` (which tool for which vector)
- Vector-to-tool mapping already implicit in each vector's try/catch
- `--install-missing` uses `npm install --save-dev` via child_process
