# Story 1.3: Detecção Automática de Solo Developer

**Epic:** UX Friction Reduction — Devflow Onboarding
**Sprint:** 1 (Curto Prazo)
**Priority:** P0 — Bloqueante
**Effort:** 2 pontos
**Status:** Done

## Problem Statement

Constitution C12 exige implementer != approver. No `gatekeep`, usuário solo é bloqueado com "Same actor cannot implement AND approve". A solução existe (`devflow config set reviewMode solo-hardened`) mas o usuário precisa saber disso. Não há detecção proativa de ambiente solo.

## User Story

AS a solo developer working alone
I WANT Devflow to detect I'm solo and offer self-approval path
SO THAT I'm not blocked by a gate designed for teams

## Acceptance Criteria

1. `gatekeep` detects same-actor scenario AND checks git config for other committers:
   - If repo has 1 committer in last 30 days → show solo warning + offer switch
   - If repo has 2+ committers → block normally (team enforcement)
2. Block message changes from "Same actor cannot implement AND approve" to:
   "🔒 C12: Independent review required. But you appear to be the only committer. Switch to solo-hardened mode? [y/N]"
3. If user accepts, auto-runs `devflow config set reviewMode solo-hardened` and retries gatekeep
4. `solo-hardened` mode still requires adversarial review evidence as compensating control
5. `devflow status` shows when in solo-hardened mode: "⚠️ Solo Mode — self-approval enabled, adversarial review required"
