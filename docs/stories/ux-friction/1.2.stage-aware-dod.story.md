# Story 1.2: Stage-Aware DoD Checks

**Epic:** UX Friction Reduction — Devflow Onboarding
**Sprint:** 1 (Curto Prazo)
**Priority:** P0 — Bloqueante
**Effort:** 3 pontos
**Status:** Done

## Problem Statement

`devflow feature complete` roda 25 checks de uma vez, incluindo checks de código (coverage, lint, typecheck, circular deps) antes mesmo de existir código. O resultado é um "banho de vermelho" que desmotiva o usuário. Checks irrelevantes ao estado atual não deveriam executar.

## User Story

AS a developer who just created feature artifacts
I WANT `feature complete` to only run checks relevant to my current state
SO THAT I get actionable feedback instead of 25 red X's

## Acceptance Criteria

1. `feature complete` detects current feature state and runs only applicable checks:
   - `feature-empty` → checks 1-4 (artifacts only)
   - `feature-requirements` → checks 1-4
   - `feature-design` → checks 1-4, 14
   - `feature-test-plan` → checks 1-4, 23
   - `feature-coding-ready` → checks 1-4, 14, 23
   - `feature-coding-in-progress` → checks 5-13, 24
   - `feature-verification` → checks 5-13, 16, 24
   - `feature-review` → checks 15, 19, 20, 25
   - `feature-done` → all 25 checks
2. Checks not yet applicable show as `⏭️ SKIPPED (requires state: <state>)` in dim text
3. Summary line shows: "N of M applicable checks passed" instead of "N of 25 passed"
4. `--all` flag forces all 25 checks regardless of state (for CI/release mode)
5. State-to-check mapping is extracted to a data structure (not hardcoded in the runner)
