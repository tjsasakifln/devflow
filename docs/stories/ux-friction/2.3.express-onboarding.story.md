# Story 2.3: Express Onboarding — `devflow quickstart`

**Epic:** UX Friction Reduction — Devflow Onboarding
**Sprint:** 2 (Médio Prazo)
**Priority:** P1 — Alta
**Effort:** 3 pontos
**Status:** Done

## Problem Statement

Superfície de comandos tem 25+ itens. Help do Commander mostra muralha de texto. Comandos essenciais (install, status, next) misturados com avançados (adversarial-review, drift check, eval run). Usuário novo não sabe por onde começar.

## User Story

AS a first-time user who hasn't read any docs
I WANT a guided 3-step interactive onboarding
SO THAT I can go from zero to understanding the tool in under 5 minutes

## Acceptance Criteria

1. `devflow quickstart` launches interactive onboarding wizard with exactly 3 steps:
   - **Step 1 "What do you have?"**: new project or existing project — auto-detected, user confirms
   - **Step 2 "What do you want?"**: build feature, understand codebase, review code — single select
   - **Step 3 "Let's go!"**: shows EXACT commands to run, in order, with one-line explanations
2. Each step shows progress: `[1/3]`, `[2/3]`, `[3/3]`
3. Quickstart auto-detects project state and pre-selects defaults
4. After step 3, offers "Run step 1 now? [Y/n]" and executes the first command
5. `devflow --help` shows `quickstart` as first command in "Getting Started" section
6. Command help reorganized into sections: Getting Started, Features, Review, Advanced
