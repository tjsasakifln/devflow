# Story 3.1: Sanity Score para Artefatos

**Epic:** UX Friction Reduction — Devflow Onboarding
**Sprint:** 3 (Longo Prazo)
**Priority:** P1 — Alta
**Effort:** 5 pontos
**Status:** InProgress

## Problem Statement

Usuário preguiçoso preenche artifacts com Lorem Ipsum ou placeholder para passar nos checks de "existe arquivo". DoD check 22 (heuristic semantic quality) existe, mas tem fallback silencioso `catch { valid = true }` — se o validador falha, o check passa. Resultado: artifacts falsos aprovados, código gerado sem qualidade.

## User Story

AS a team lead who wants real quality assurance
I WANT Devflow to measure actual content density in artifacts
SO THAT placeholder-filled templates are rejected before code is written

## Acceptance Criteria

1. New `sanity-score` check that measures artifact quality across 4 metrics:
   - **Content density**: ratio of user-written content vs template boilerplate (>40% required)
   - **Section completion**: % of sections with >30 chars of non-placeholder text (>60% required)
   - **Specificity score**: presence of concrete names, values, file paths (not generic terms)
   - **Placeholder detection**: flags "TODO", "TBD", "n/a", "lorem ipsum", "test", "placeholder"
2. Sanity score shown as 0-100 with color coding: 80-100 green, 50-79 yellow, 0-49 red
3. Score <50 blocks `feature prompt` generation
4. Check 22 (heuristic semantic quality) is REPLACED by sanity score (not duplicated)
5. Validator failure NEVER silently passes — if unavailable, shows ADVISORY instead of PASS
6. `devflow doctor` includes sanity score in artifact health section
7. Standalone `devflow sanity-score <featureId>` command available
