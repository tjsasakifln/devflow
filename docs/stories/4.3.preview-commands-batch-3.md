## Dev Notes

- `adversarial-review-ai` é o único comando que depende de LLM externo — fallback obrigatório
- `trace` usa dados já existentes nos audit logs — é mais renderização que lógica nova
- `promote` é o mais arriscado: mexer em ambiente requer confirmação explícita
- Remover tier system impacta `src/main.ts`, `src/cli/index.ts`, e possivelmente `devflow next`
- Fix pre-existing: `src/kernel/workflow/loader.ts` — `js-yaml` import broken (default export) — fixed alongside story work to enable CLI tests

## Risks

Médio — `adversarial-review-ai` depende de LLM provider (custo, latência). Mitigação: fallback para determinístico se provider indisponível. `promote` requer CI integration (se CI não configurado, warn-and-proceed).
