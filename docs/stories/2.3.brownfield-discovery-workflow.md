## Story

As a **developer**,
I want **`devflow discover` to execute a complete brownfield analysis workflow**,
so that **I can understand any legacy codebase without manual documentation**.

## Scope

**IN:** Workflow brownfield completo (scout → archaeologist → detective → architect → writer), `devflow discover` renovado, output em `_devflow/discovery/`, flag `--phase`.
**OUT:** Reversa skills (integração como provider é opcional — funciona standalone).

**Complexity:** L (5 fases, cada uma com lógica própria. Se escopo estourar, split: 2.3a scout+archaeologist+detective, 2.3b architect+writer)

## Acceptance Criteria

1. Workflow brownfield implementado: scout → archaeologist → detective → architect → writer
2. `devflow discover` orquestra as fases sequencialmente
3. Output em `_devflow/discovery/`: system-architecture.md, SCHEMA.md, technical-debt.md, TECHNICAL-DEBT-REPORT.md
4. Cada fase pode rodar independente: `devflow discover --phase=scout`
5. Integração com Reversa skills como providers opcionais

## Integration Verification

1. IV1: Rodar `devflow discover` no próprio repo Devflow → relatórios gerados
2. IV2: Conteúdo dos relatórios referenceável por stories subsequentes
3. IV3: `devflow next` após discover recomenda criar epic/story baseado nos findings

## Dev Notes

- `src/commands/discover.ts` (815 LOC) já existe como experimental — expandir, não reescrever do zero
- Fases são sequenciais: output de uma é input da próxima
- Flag `--phase` permite retomar de fase específica (útil para debugging)
- Reversa skills em `.claude/skills/reversa/` são providers opcionais — discover funciona standalone
- Cada fase deve gerar output intermediário para checkpoint/resume

## Risks

Médio — 5 fases ambicioso para uma story. Scope creep possível. Mitigação: flag `--phase` permite entrega incremental. Cada fase pode ser validada independentemente.
